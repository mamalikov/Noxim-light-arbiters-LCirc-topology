LCirc_noxim – Noxim with L‑type Circulant Topology and Advanced Arbitration
============================================================================

This repository is a research fork of **Noxim**, the Network‑on‑Chip simulator developed at the University of Catania, extended with:

- additional **arbitration (selection) strategies** for Mesh NoCs,
- a new **L‑type circulant (LCirc) topology**,
- several **routing algorithms** for LCirc,
- automation and analysis scripts for large experimental campaigns.

The original Noxim codebase, build system and license remain unchanged; this fork adds functionality on top of it.

If you use this fork in your research, please cite the original Noxim papers (see below) and, if possible, reference this extended version as well.


Overview of Extensions
----------------------

### 1. New arbitration (selection) strategies

For both Mesh and LCirc topologies, we compare the following arbitration strategies:

- `HOPCOUNT_MAX` – priority to packets with **maximum** hop count.
- `HOPCOUNT_MIN` – priority to packets with **minimum** hop count.
- `DISTANCE_MIN` – priority to packets with **minimum remaining distance**.
- `DISTANCE_MAX` – priority to packets with **maximum remaining distance**.

For **Mesh**:
- Remaining distance is computed as Manhattan distance in \((x, y)\) coordinates:
  \[
  |x{dst} - x{src}| + |y{dst} - y{src}|
  \]
- Distances are cached in a pre‑computed all‑pairs table for each mesh size (4×4, 8×8), so lookups are \(O(1)\).


### 2. L‑type Circulant (LCirc) topology and routing

This fork adds support for an **L‑type circulant** topology and several routing algorithms for it. At a high level:

- LCirc is a 2D circulant graph with two generating strides (s1, s2), which is placed into mesh structure.
- The following routing algorithms are supported (see `plot_results.py` and routing sources):
  - `LCIRC` – baseline LCirc routing (route‑zero3‑style).
  - `LCIRC_DIJKSTRA` – LCirc routing based on pre‑computed shortest paths (Dijkstra).
  - `LCIRC_VIRTUALCOORD` – LCirc routing in a virtual coordinate system.
  - `LCIRC_SIMPLE` – simple clockwise traversal.

Configuration files for LCirc topologies are provided under `config_examples/` (see the LCirc‑related YAML files).


### 3. Mesh experiments and comparison scripts

This fork adds a complete workflow to:

1. Run Mesh experiments for different arbiters, routing algorithms, traffic patterns and PIR.
2. Aggregate and post‑process results.
3. Plot both **absolute metrics** and **relative differences vs RR**.
4. Analyse **win ratios** of arbiters in PIR ranges.

#### 3.1 Running Mesh simulations

Mesh experiments are driven by:

- `run_arbitration_comparison.sh`

Key characteristics:

- Topology: Mesh 4×4 and Mesh 8×8 (no HUB).
- Config files:
  - `config_examples/default_configMeshNoHUB.yaml` (4×4)
  - `config_examples/default_configMeshNoHUB_8x8.yaml` (8×8)
- Routing algorithms (command‑line `-routing`):
  - `XY`, `WEST_FIRST`, `NORTH_LAST`, `NEGATIVE_FIRST`, `ODD_EVEN`
- Arbitration strategies:
  - `RANDOM`, `HOPCOUNT_MAX`, `HOPCOUNT_MIN`, `DISTANCE_MIN`, `DISTANCE_MAX`
- Traffic patterns:
  - `TRAFFIC_RANDOM`, `TRAFFIC_TRANSPOSE1`, `TRAFFIC_BIT_REVERSAL`,
    `TRAFFIC_SHUFFLE`, `TRAFFIC_BUTTERFLY`
- PIR values:
  - `0.01` to `0.10` with step `0.01`.
- For each `(mesh size, routing, traffic, PIR, arbiter)`:
  - multiple successful runs are executed,
  - statistics are averaged,
  - results are written to:
    - `results/mesh4x4/<ROUTING>/TRAFFIC_*/mesh_arbitration_PIR_*.txt`
    - `results/mesh8x8/<ROUTING>/TRAFFIC_*/mesh_arbitration_PIR_*.txt`

To launch the full Mesh campaign:

```bash
./run_arbitration_comparison.sh
```


#### 3.2 Plotting Mesh results

Mesh arbitration plots are generated by:

- `plot_mesh_arbitration_results.py`

Main features:

- Plots **relative difference vs RR** or **absolute values**:
  - Relative mode (default): (RR - ARB) / RR * 100%
    - For throughput, sign is inverted so that negative values correspond to improvements.
  - Absolute mode (`--absolute`): plots metric values directly (including RR as a separate curve).
- Supports both Mesh sizes:
  - `--mesh {4x4, 8x8}`
- Flexible aggregation:
  - Specify `--routing` and `--traffic` for a specific combination.
  - Omit `--traffic` to **average over all traffic patterns** for a given routing.
  - Omit `--routing` (but give `--traffic`) to **average over all routing algorithms**.
  - Omit both `--routing` and `--traffic` to average over **all routing and traffic** for Mesh 4×4 and 8×8.
- PIR filters:
  - `--pir-min`, `--pir-max` to restrict the PIR range along X.

Examples:

```bash
# Relative comparison for Mesh 4x4, XY routing, TRAFFIC_RANDOM
python plot_mesh_arbitration_results.py max_delay --mesh 4x4 --routing XY --traffic TRAFFIC_RANDOM

# Absolute max_delay for all routing/traffic in PIR range [0.04, 0.07]
python plot_mesh_arbitration_results.py max_delay --pir-min 0.04 --pir-max 0.07 --absolute
```


#### 3.3 Mesh arbitration analysis scripts

Additional analysis helpers:

- `analyze_mesh_arbitration.py`
  - Computes, for Mesh 4×4 and 8×8:
    - which arbiter wins most often per metric (throughput, avg_delay, max_delay),
    - breakdown by PIR ranges (`low` 0.01–0.03, `medium` 0.04–0.07, `high` 0.08–0.10),
    - per routing, per traffic pattern,
    - average relative improvements vs RR.


### 4. LCirc experiments

LCirc (L‑type circulant) experiments are driven by:

- `run_lcirc_comparison.sh`

This script runs a set of LCirc configurations and routing algorithms, collects the same metrics as for Mesh, and writes comparison results into a text file:

- `lcirc_comparison_results.txt`

Post‑processing for LCirc:

- `plot_results.py`
  - Plots **per‑algorithm LCirc results** vs PIR (absolute metrics).
  - Uses `PARAMETERS` and `ALGORITHM_SUFFIXES` to map internal algorithm keys to readable labels.


Building and Running
--------------------

This fork keeps the original Noxim build system. In short:

```bash
cd bin
make        # builds the noxim binary
```

Make sure SystemC and other dependencies are installed as required by the original project.
See the upstream Noxim documentation for detailed build instructions and troubleshooting.

After building:

- Use `run_arbitration_comparison.sh` for Mesh experiments.
- Use `run_lcirc_comparison.sh` for LCirc experiments.
